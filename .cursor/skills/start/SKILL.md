# Start — Инициализация проекта

## Inputs

- [project-state.json](../../../project-state.json) (корень проекта)
- Папки: [sources/](../../../sources/), [workspace/](../../../workspace/), [content/](../../../content/)

## Logic

### Шаг 1: Проверка состояния

Прочитай [project-state.json](../../../project-state.json). Если файл не существует — создай его по шаблону (см. Outputs). Проверь секцию `skills` для понимания, на каком этапе остановился проект.

### Шаг 2: Сканирование структуры

Проверь наличие и содержимое ключевых папок:

```
sources/
  original/
  converted/
workspace/
  brief.md
  course-structure.yaml
  sources-report.md
  audit/
content/
  pages/
  files/
```

Если каких-то папок нет — создай пустую структуру.

### Шаг 3: Определение статуса проекта

**Существующий проект** — одно из условий истинно:

- если в [project-state.json](../../../project-state.json) указаны статусы
- если в [content/pages/](../../../content/pages/) есть `.yaml` файлы (даже без наличия статусов)

**Новый проект** — ни одно из условий выше не выполнено.

### Шаг 4: Маршрутизация

**Если существующий проект:**

Если в файле [project-state.json](../../../project-state.json) указаны статусы, кратко сообщи пользователю состояние проекта и предложи продолжить либо сделать что-то еще.

Если статусов нет, предложи внести правки в контент.

**Если новый проект:**

1. Проверить [sources/converted/](../../../sources/converted/) на наличие `.md` файлов.
2. Проверить [sources/original/](../../../sources/original/) на наличие исходных файлов (.docx, .pdf, .pptx, .xlsx и др.).

Логика выбора:

- **converted/ содержит .md** → `source_mode: full`, направить на `analysis-scanner`
- **converted/ пуста, original/ содержит файлы** → `source_mode: full`, направить на `analysis-scanner` (он сконвертирует)
- **Обе папки пусты** → предложить три варианта (см. ниже)

**Если обе папки пусты**, предложи пользователю:
1. **Загрузить исходники** — «Положите файлы в [sources/original/](../../../sources/original/) и скажите "готово"»
2. **Работать без исходников** — «Я сам сгенерирую контент на основе брифования. Задам больше вопросов, чтобы понять тему.» → `source_mode: none`, направить на `analysis-router` (сканер не нужен)
3. **Частичные исходники позже** — «Начнём брифование, а исходники добавите по ходу дела» → `source_mode: partial`, направить на `analysis-router`

Запиши выбранный `source_mode` в [project-state.json](../../../project-state.json).

### Шаг 5: Обновление [project-state.json](../../../project-state.json) и подтверждение

1. Обнови [project-state.json](../../../project-state.json): отметь `start` → `completed`.
2. Автоматически переходи к следующему скиллу.

## Outputs

- Сообщение пользователю о статусе проекта
- Обновлённый [project-state.json](../../../project-state.json) (включая `source_mode`)
- Направление на скиллы `analysis-scanner` (если есть исходники) или `analysis-router` (если исходников нет), либо скиллы других кластеров для существующих проектов

Шаблон [project-state.json](../../../project-state.json) → [`assets/project-state-template.json`](assets/project-state-template.json)
