# Quality-Fix — Исправление по отчётам аудита

## Inputs

- [project-state.json](../../../project-state.json) (статус проекта)
- [workspace/audit/](../../../workspace/audit/) (папка с отчётами аудита — по файлу на каждую проверку)
- [content/pages/*.yaml](../../../content/pages/) (файлы страниц, которые нужно исправить)
- [workspace/brief.md](../../../workspace/brief.md) (для контекста при исправлениях)
- [sources/converted/*.md](../../../sources/converted/) (для сверки с исходниками)

## Logic

### Шаг 0: Проверка статуса проекта

Прочитай [project-state.json](../../../project-state.json). Убедись, что скилл `quality-audit` выполнен.

### Шаг 1: Чтение отчётов аудита

Прочитай все YAML-файлы в [workspace/audit/](../../../workspace/audit/). Каждый файл — отчёт отдельной проверки (completeness, compliance, methodology, accuracy, technical). Собери все `issues` из всех файлов. Отсортируй по severity: critical → major → minor.

### Шаг 2: Исправление проблем

Для каждой записи со `status: to_fix`:

1. Открой файл страницы из [content/pages/](../../../content/pages/), соответствующий `page_id`
2. Найди блок по `block_id` (если указан)
3. Прочитай `description` и `suggestion` из отчёта
4. Внеси исправление:

**Для проблем из `completeness.yaml`:**
- Перепиши или дополни текст блока
- Убедись, что новый текст соответствует [brief.md](../../../workspace/brief.md)

**Для проблем из `compliance.yaml`:**
- Проверь соответствие брифу (tone of voice, обращение, уровень аудитории, редполитика)
- Исправь формулировки и стиль без изменения фактического смысла

**Для проблем из `methodology.yaml`:**
- Скорректируй подход (добавь пример, измени структуру, усиль объяснение)
- Проверь соответствие learning objectives

**Для проблем из `accuracy.yaml`:**
- Сверь с исходниками в [sources/converted/](../../../sources/converted/)
- Исправь фактические ошибки
- Убедись, что нет выдуманного контента

**Для проблем из `technical.yaml`:**
- Исправь синтаксис YAML
- Заполни пустые обязательные поля
- Устрани дубликаты ID

### Шаг 3: Обновление отчётов

После исправления каждой проблемы обнови запись в исходном файле отчёта в [workspace/audit/](../../../workspace/audit/):

```yaml
- issue_id: CMP_001
  # ... остальные поля без изменений ...
  status: fixed
  fixed_at: "[ISO timestamp]"
  fix_description: >-
    [Краткое описание того, что было сделано]
```

### Шаг 4: Повторная проверка

После исправления всех пунктов:

1. Пересчитай статистику в `summary` внутри каждого обновлённого отчёта
2. Проверь, не породили ли исправления новых проблем (каскадный эффект)
3. Если обнаружены новые проблемы — добавь их в соответствующий файл отчёта со `status: to_fix`

### Шаг 5: Обновление [project-state.json](../../../project-state.json) и подтверждение

1. Обнови [project-state.json](../../../project-state.json): отметь `quality-fix` → `completed`.
2. Покажи сводку:
   - Исправлено проблем: N из M
   - Критических исправлено: N
   - Серьёзных исправлено: N
   - Мелких исправлено: N
   - Новых проблем обнаружено: N (если есть)
3. Если все проблемы исправлены — сообщи, что курс прошёл аудит.
4. Если остались нерешённые или новые проблемы — предложи повторный `quality-audit`.
5. Спроси подтверждение. Не переходи без явного согласия пользователя.

## Outputs

- Обновлённые файлы [content/pages/*.yaml](../../../content/pages/) с исправлениями
- Обновлённые файлы в [workspace/audit/](../../../workspace/audit/) (статусы `to_fix` → `fixed`)
- Обновлённый [project-state.json](../../../project-state.json)
- Итоговый отчёт о проделанной работе
