# Generation-Image — Оркестрация визуального контента

## Inputs

- [project-state.json](../../../project-state.json) (статус проекта)
- [workspace/brief.md](../../../workspace/brief.md) (стилистика, бренд)
- [content/pages/*.yaml](../../../content/pages/) (файлы страниц)
- [content/files/](../../../content/files/) (готовые изображения, если есть)

> **Важно:** Этот скилл НЕ генерирует промты сам. Он выступает оркестратором: собирает контекст и запускает субагент `generator-images` для каждой страницы, у которой есть незаполненные image/hotspot-блоки.

## Субагент

| Файл | Режим | Назначение |
|------|-------|------------|
| [`.cursor/agents/generator-images.md`](../../agents/generator-images.md) | background | Генерация промтов и заполнение image/hotspot-блоков для одной страницы |

## Logic

### Шаг 0: Проверка статуса проекта

Прочитай [project-state.json](../../../project-state.json).

1. Убедись, что скилл `generation-quiz` выполнен.
2. Проверь `_active`: если установлен другим скиллом — предупреди пользователя и не начинай без подтверждения. Если `_active` от предыдущего прерванного запуска этого же скилла — подхвати работу (см. «Восстановление после сбоя» в Шаге 4).
3. Установи `_active`:

```yaml
_active:
  skill: generation-image
  started: "{ISO timestamp}"
  pages_total: {количество страниц с empty-блоками}
  pages_done: 0
```

### Шаг 1: Подготовка контекста

1. Прочитай [workspace/brief.md](../../../workspace/brief.md) — извлеки блок с информацией о стилистике, бренд-цветах, tone of voice. Сформируй краткое описание (5–10 строк) — **brand_context**.
2. Просканируй [content/files/](../../../content/files/) на наличие готовых изображений (jpg, png, svg, webp). Составь список — **available_images**.

### Шаг 2: Определение страниц для обработки

Просканируй все файлы [content/pages/*.yaml](../../../content/pages/). Для каждой страницы обойди все фреймы (включая вложенные сцены `nested_scene` и элементы `accordion`, `tabs`, `carousel`) и проверь, есть ли блоки с `block: image` или `block: hotspot` со `status: empty`.

Составь список **pages_to_process** — только те страницы, где есть хотя бы один такой блок.

Если список пуст — сообщи пользователю, что все image/hotspot-блоки уже заполнены, обнови [project-state.json](../../../project-state.json) и заверши работу.

### Шаг 3: Блокировка и запуск субагентов

**3a. Установка блокировок.** Для каждой страницы из `pages_to_process` добавь в файл страницы:

```yaml
_lock:
  agent: generator-images
  started: "{ISO timestamp}"
```

**3b. Запуск субагентов.** Для каждой страницы запусти субагент `generator-images` **в фоне** с промтом. Каталог страниц: [content/pages/](../../../content/pages/), медиа: [content/files/](../../../content/files/).

```
Обработай image/hotspot-блоки в файле страницы.

Файл страницы: content/pages/{filename}

Бренд-контекст:
{brand_context}

Доступные изображения в content/files/:
{available_images — список файлов или "нет готовых изображений"}

Правила генерации промтов:
- Стиль: flat, isometric, realistic, cartoon, corporate — выбирай исходя из контекста курса
- Описывай конкретные объекты и расположение
- Цветовая палитра: {цвета из brand_context}
- Соотношение сторон: 16:9
- Без текста на изображениях
- Промт на английском языке
- Контекст берётся из purpose блока и текстовых блоков страницы
```

Все субагенты запускаются **параллельно** в одном сообщении.

### Шаг 4: Валидация и снятие блокировок

Дождись завершения всех субагентов. Для каждой обработанной страницы:

1. **Валидация:** проверь, что YAML парсится, все целевые image/hotspot-блоки имеют `status: filled` или `prompt_ready`, обязательные поля заполнены.
2. **Если валидация пройдена:** удали `_lock` из файла страницы, обнови `_active.pages_done`.
3. **Если валидация не пройдена:** перезапусти субагент для этой конкретной страницы (одна попытка). Если повторная валидация снова провалилась — сними `_lock`, добавь страницу в список ошибок.

**Восстановление после сбоя:** если при запуске обнаружен `_active` от предыдущего прерванного запуска — сними все устаревшие `_lock`, составь новый `pages_to_process` из страниц с `empty`-блоками.

### Шаг 5: Общий отчёт по всей цепочке генерации

Это финальный этап цепочки `generation-text → generation-quiz → generation-image`.

1. Удали `_active` из [project-state.json](../../../project-state.json).
2. Обнови [project-state.json](../../../project-state.json): отметь `generation-image` → `completed`.
3. Прочитай [project-state.json](../../../project-state.json) — собери данные о выполнении всех трёх этапов.
4. Покажи **общий отчёт по всей генерации**:

```
═══ Генерация контента завершена ═══

📝 Тексты (generation-text):
   Страниц обработано: N
   Блоков заполнено: N

📋 Проверочные задания (generation-quiz):
   Страниц обработано: N
   Блоков заполнено: N

🖼 Изображения (generation-image):
   Привязано готовых: N
   Сгенерировано промтов: N
   Hotspot-блоков: N

⚠ Ошибки: N (если есть — перечисли)
```

5. Предложи:
   - Запустить `quality-audit` для проверки курса
   - Сгенерировать изображения по промтам и добавить в [content/files/](../../../content/files/)
   - Проверить результат вручную
6. Спроси подтверждение. Не переходи без явного согласия.

## Outputs

- Обновлённые файлы [content/pages/*.yaml](../../../content/pages/) (image-блоки: `filled` или `prompt_ready`; hotspot-блоки: `filled`)
- Обновлённый [project-state.json](../../../project-state.json)
- Общий отчёт по трём этапам генерации
- Направление на `quality-audit`
