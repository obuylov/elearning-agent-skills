# Edit — Точечные правки в курсе

## Inputs

- [project-state.json](../../../project-state.json) (статус проекта)
- [content/pages/*.yaml](../../../content/pages/) (файлы страниц курса)
- [workspace/brief.md](../../../workspace/brief.md) (для контекста: тон, аудитория, принципы)
- [workspace/course-structure.yaml](../../../workspace/course-structure.yaml) (для понимания структуры курса)
- [sources/converted/*.md](../../../sources/converted/) (если нужна сверка с исходниками)

## Когда использовать

Этот скилл — точка входа для любых правок после генерации контента. Используй, когда пользователь:
- Просит изменить текст на конкретной странице
- Хочет переформулировать вопрос квиза
- Просит добавить, удалить или переместить блок
- Хочет поменять тип блока (например, tinymce → accordion)
- Просит изменить тон, стиль или объём конкретного фрагмента
- Хочет добавить новую страницу в курс
- Просит обновить промт для изображения

Этот скилл **не** запускается автоматически в пайплайне. Он вызывается пользователем напрямую в любой момент после `course-template`.

## Logic

### Шаг 0: Проверка блокировок и понимание запроса

**Проверка `_active` в [project-state.json](../../../project-state.json):**
Если установлен `_active` (другой скилл сейчас обрабатывает страницы) — предупреди пользователя:
> «Сейчас выполняется {_active.skill} (обработано {pages_done}/{pages_total} страниц). Рекомендую дождаться завершения. Продолжить правки на свой риск?»

Если пользователь не подтверждает — заверши работу.

**Определение запроса.** Определи, что именно хочет пользователь. Если запрос нечёткий — уточни:
- Какую страницу / блок нужно изменить?
- Что именно не устраивает?
- Каким должен быть результат?

Если пользователь не знает ID страницы или блока — покажи список страниц из [content/pages/](../../../content/pages/) с их названиями и предложи выбрать.

**Проверка `_lock` на целевой странице:**
Прочитай файл выбранной страницы. Если есть `_lock` — откажись редактировать:
> «Страница {filename} сейчас обрабатывается субагентом {_lock.agent}. Выберите другую страницу или дождитесь завершения.»

### Шаг 1: Внесение правок

Прочитай файл страницы, найди нужный блок. Блоки находятся внутри фреймов (`page.scene.frames[].blocks[]`), а также внутри вложенных сцен и банков вопросов. Структура по модели → [`2_1_course-model`](../2_1_course-model/SKILL.md).

**Типы правок:**

**Редактирование контента блока:**
- Перепиши или скорректируй `content`, `question`, `options`, `prompt` и т.д.
- Сохрани `status: filled` (или `prompt_ready` для image)
- Убедись, что правка соответствует [brief.md](../../../workspace/brief.md) (тон, аудитория)

**Добавление нового блока:**
- Определи, в какой фрейм добавить (по `role` фрейма)
- Сгенерируй уникальный `id` — 7-символьный hex. Убедись, что он не дублирует существующие ID в курсе
- Укажи `type` (`presentation` / `assessment`) и `block` (конкретный тип)
- Заполни контент сразу (не оставляй `status: empty`)

**Удаление блока:**
- Удали блок из фрейма в YAML-файле страницы
- Если блок был assessment и это влияет на `pass_criteria` — предупреди пользователя
- Если фрейм остаётся пустым — удали и фрейм

**Смена типа блока:**
- Пересоздай блок с новой структурой, сохранив `id` и `purpose`
- Перегенерируй контент под новый формат

**Добавление нового фрейма:**
- Определи позицию в сцене (после какого фрейма)
- Задай `role` и `completion`
- Сгенерируй уникальный `id`

**Добавление новой страницы:**
- Определи позицию в курсе (после какой страницы)
- Создай файл в [content/pages/](../../../content/pages/) с правильным именованием
- Структура: `page → scene (vertical_scene) → frames → blocks`
- Заполни все блоки контентом сразу
- Обнови [workspace/course-structure.yaml](../../../workspace/course-structure.yaml)

**Перемещение блока или страницы:**
- Обнови порядок в файле / файлах
- Проверь, не сломались ли мостики между страницами (вступления/выводы)

### Шаг 2: Проверка согласованности

После внесения правок проверь:
- Не нарушилась ли логическая последовательность (если правка затронула вступление или вывод)
- Корректны ли ссылки между страницами (мостики)
- Не противоречит ли изменение другим блокам на странице
- Соответствует ли тон и стиль [brief.md](../../../workspace/brief.md)

Если правка каскадно влияет на соседние блоки или страницы — предупреди пользователя и предложи скорректировать затронутые места.

### Шаг 3: Показ результата

Покажи пользователю изменённый фрагмент. Если правка масштабная — покажи всю страницу.

Спроси: «Так хорошо? Нужно ещё что-то поправить?»

Если пользователь хочет ещё правки — вернись к Шагу 0.

### Шаг 4: Завершение

Когда пользователь доволен:
- Убедись, что все изменённые файлы сохранены
- Если были структурные изменения (новые/удалённые страницы) — обнови [workspace/course-structure.yaml](../../../workspace/course-structure.yaml)
- Сообщи, что правки внесены

> **Примечание:** Этот скилл не обновляет [project-state.json](../../../project-state.json) — он работает вне основного пайплайна. Пользователь может вызывать его многократно.

## Outputs

- Обновлённые файлы [content/pages/*.yaml](../../../content/pages/) с правками
- При структурных изменениях — обновлённый [workspace/course-structure.yaml](../../../workspace/course-structure.yaml)
