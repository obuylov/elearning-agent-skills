# Generation-Quiz — Оркестрация генерации проверочных заданий

## Inputs

- [project-state.json](../../../project-state.json) (статус проекта)
- [workspace/brief.md](../../../workspace/brief.md) (бизнес-драйвер, принципы проектирования, стратегия оценки)
- [content/pages/*.yaml](../../../content/pages/) (файлы страниц)

> **Важно:** Этот скилл НЕ генерирует квизы сам. Он выступает оркестратором: собирает контекст и запускает субагент `generator-quiz` для каждой страницы с незаполненными assessment-блоками.

## Субагент

| Файл | Режим | Назначение |
|------|-------|------------|
| [`.cursor/agents/generator-quiz.md`](../../agents/generator-quiz.md) | background | Генерация assessment-блоков для одной страницы |

## Logic

### Шаг 0: Проверка статуса проекта

Прочитай [project-state.json](../../../project-state.json).

1. Убедись, что скилл `generation-text` выполнен.
2. Проверь `_active`: если установлен другим скиллом — предупреди пользователя и не начинай без подтверждения. Если `_active` от предыдущего прерванного запуска этого же скилла — подхвати работу (см. «Восстановление после сбоя» в Шаге 4).
3. Установи `_active`:

```yaml
_active:
  skill: generation-quiz
  started: "{ISO timestamp}"
  pages_total: {количество страниц с empty-блоками}
  pages_done: 0
```

### Шаг 1: Определение стратегии тестирования

Из [brief.md](../../../workspace/brief.md) определи:
- Бизнес-драйвер (compliance → жёсткий контроль, knowledge → на понимание, performance → сценарные вопросы)
- Уровень сложности вопросов
- Нужен ли пороговый балл

Сформируй краткое описание — **testing_strategy**.

### Шаг 2: Определение страниц для обработки

Просканируй все файлы [content/pages/*.yaml](../../../content/pages/). Для каждой страницы обойди все фреймы (включая фреймы внутри `question_bank`) и проверь, есть ли assessment-блоки со `status: empty`:
- `single_choice`, `multiple_choice`, `sorting`, `matching`, `sequence`, `classify`, `fill_gap`, `scenario`, `case_question`, `reflection`

Составь список **pages_to_process** — только те страницы, где есть хотя бы один такой блок.

Если список пуст — сообщи пользователю, обнови [project-state.json](../../../project-state.json) и заверши работу.

### Шаг 3: Блокировка и запуск субагентов

**3a. Установка блокировок.** Для каждой страницы из `pages_to_process` добавь в файл страницы:

```yaml
_lock:
  agent: generator-quiz
  started: "{ISO timestamp}"
```

**3b. Запуск субагентов.** Для каждой страницы запусти субагент `generator-quiz` **в фоне** с промтом:

```
Заполни assessment-блоки на странице.

Файл страницы: content/pages/{filename}

Стратегия тестирования:
{testing_strategy}

Правила:
- Вопросы проверяют понимание, а не запоминание формулировок
- Дистракторы должны быть правдоподобными
- Feedback — содержательная обратная связь, не просто «Правильно!» / «Неправильно!»
- Задания опираются на контент presentation-блоков (status: filled) той же страницы
- Assessment-блоки находятся в фреймах (role: practice) и внутри банков вопросов (question_bank)
```

Все субагенты запускаются **параллельно** в одном сообщении.

### Шаг 4: Валидация и снятие блокировок

Дождись завершения всех субагентов. Для каждой обработанной страницы:

1. **Валидация:** проверь, что YAML парсится, все целевые assessment-блоки имеют `status: filled`, обязательные поля заполнены.
2. **Если валидация пройдена:** удали `_lock` из файла страницы, обнови `_active.pages_done`.
3. **Если валидация не пройдена:** перезапусти субагент для этой страницы (одна попытка). Если повторная валидация провалилась — сними `_lock`, добавь страницу в список ошибок.

**Восстановление после сбоя:** если при запуске обнаружен `_active` от предыдущего прерванного запуска — сними все устаревшие `_lock`, составь новый `pages_to_process` из страниц с `empty`-блоками.

### Шаг 5: Завершение и автозапуск следующего этапа

1. Удали `_active` из [project-state.json](../../../project-state.json).
2. Обнови [project-state.json](../../../project-state.json): отметь `generation-quiz` → `completed`.
3. Кратко сообщи пользователю прогресс (1–2 строки): «Квизы готовы (N страниц, N блоков). Перехожу к генерации изображений...»
4. **Немедленно запусти** скилл `generation-image`. Не спрашивай подтверждения — это единая цепочка генерации.

## Outputs

- Обновлённые файлы [content/pages/*.yaml](../../../content/pages/) с заполненными assessment-блоками (`status: filled`)
- Обновлённый [project-state.json](../../../project-state.json)
- **Автоматический запуск** `generation-image`
