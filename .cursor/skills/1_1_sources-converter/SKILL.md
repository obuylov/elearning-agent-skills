# Sources-Converter — Конвертация исходников

## Inputs

- [project-state.json](../../../project-state.json) (статус проекта)
- Файлы в [sources/original/](../../../sources/original/) (.docx, .pdf, .pptx, .xlsx и др.)

## Logic

### Шаг 1: Конвертация исходников

Проверь папки [sources/original/](../../../sources/original/) и [sources/converted/](../../../sources/converted/).

**Если в [sources/original/](../../../sources/original/) есть файлы, а [sources/converted/](../../../sources/converted/) пуста или не содержит соответствующих .md файлов** — выполни конвертацию:

Запусти скрипт конвертации из этого скилла:

```bash
python .cursor/skills/1_1_sources-converter/scripts/convert.py
```

Скрипт автоматически:
- Найдёт все поддерживаемые файлы в `sources/original/` **включая подпапки** (рекурсивный обход)
- Установит `markitdown` при необходимости
- Сконвертирует каждый файл в `.md` и сохранит в `sources/converted/`
- **Сохранит структуру подпапок:** `sources/original/модуль_1/doc.docx` → `sources/converted/модуль_1/doc.md`
- Пропустит файлы, для которых `.md` уже существует (используй `--force` для переконвертации)

Поддерживаемые форматы: PDF, Word, PowerPoint, Excel, CSV, JSON, XML, HTML, EPUB, изображения, аудио и архивы (см. список в `scripts/convert.py`).

Если конвертация завершилась ошибкой — сообщи пользователю и предложи:
- Проверить формат файла
- Установить дополнительные зависимости: `pip install "markitdown[all]"`
- Сконвертировать проблемный файл вручную

### Шаг 2: Валидация качества конвертации

Скрипт `convert.py` автоматически выводит метрики качества для каждого файла: количество слов, заголовков, наличие таблиц и мусорных символов. Используй эти метрики как основу для оценки.

**Критерии качества:**

| Качество | Условие | Действие |
|----------|---------|----------|
| `good` | Текст чистый, структура сохранена | Файл готов к анализу |
| `degraded` | Битые таблицы или потеря заголовков в большом файле | Файл пригоден, но с оговорками |
| `poor` | Мусорные символы, OCR-артефакты | Предложить ручную конвертацию |
| `empty` | < 50 слов | Предложить ручную конвертацию |

**Если есть проблемные файлы (`poor` / `empty`)** — покажи пользователю сводку разом:

> «N файлов сконвертированы с проблемами:
> - `файл1.md` — пустой (3 слова)
> - `файл2.md` — мусорные символы
>
> Варианты: (1) сконвертировать вручную, (2) предоставить в другом формате, (3) продолжить без этих файлов.»

**Если все файлы оказались `poor` или `empty`** — предложи два варианта:
1. Предоставить файлы в другом формате или сконвертировать вручную
2. Переключиться в режим `source_mode: none` (работать без исходников, на основе брифования)

Если пользователь не может предоставить файлы лучше — продолжай работать с тем, что есть (файлы `degraded` пригодны, файлы `poor` используй с предупреждением).

### Outputs

- Сконвертированные `.md` файлы в [sources/converted/](../../../sources/converted/) (с сохранением структуры подпапок)
- Консольный лог с метриками качества для каждого файла
